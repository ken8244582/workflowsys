### B001 - 端到端流程数据使用文件存储，生产环境数据丢失风险

- **模块**: 端到端流程
- **严重程度**: 高
- **状态**: 已修复
- **发现日期**: 2026-06-10
- **修复日期**: 2026-06-10

**问题描述**:
端到端流程（`e2e-processes.json` / `e2e-plans.json`）和梳理计划使用 `e2e-store.ts` 进行本地文件读写，而非像职能流程数据那样存入 Supabase 数据库。生产环境中数据写入 `/tmp/e2e-data/` 目录，该目录为临时目录，系统可能定期清理导致数据丢失。

**复现步骤**:
1. 在生产环境创建/修改端到端流程数据
2. 服务重启或系统清理 `/tmp` 目录
3. 数据丢失，回退到初始状态

**影响范围**:
- 端到端流程管理 `/e2e/list`
- 端到端梳理计划 `/e2e/plan`
- 端到端概览统计 `/e2e/overview`

**修复方案**:
将 `e2e-store.ts` 中的文件读写改为 Supabase 数据库操作，新建 `e2e_processes` 和 `e2e_plans` 两张表，与 flows / revision_plans 等表保持一致的持久化策略。

### B002 - JWT 密钥硬编码，生产环境安全隐患

- **模块**: 系统管理 - 认证
- **严重程度**: 高
- **状态**: 已修复
- **发现日期**: 2026-06-10
- **修复日期**: 2026-06-10

**问题描述**:
`src/lib/auth.ts` 中 JWT 签名密钥使用硬编码的默认值：
```typescript
const JWT_SECRET = new TextEncoder().encode(
  process.env.JWT_SECRET || 'flow-management-platform-secret-key-2024'
);
```
如果未配置环境变量 `JWT_SECRET`，则所有环境共享同一密钥，存在 token 伪造风险。

**复现步骤**:
1. 检查 `src/lib/auth.ts` 第4行
2. 确认 `process.env.JWT_SECRET` 未在部署环境中配置

**影响范围**:
- 所有依赖 JWT 认证的 API 接口
- 用户登录态可被伪造

**修复方案**:
1. 移除硬编码默认值，启动时检测 `JWT_SECRET` 未配置则返回 500 错误
2. 改为懒加载 `getSecretKey()` 函数，支持 `.env.local` 热加载
3. 已在 `.env.local` 中配置强随机密钥

### B003 - 流程清单搜索接口存在 SQL 注入风险

- **模块**: 职能流程 - 流程清单
- **严重程度**: 中
- **状态**: 已修复
- **发现日期**: 2026-06-10
- **修复日期**: 2026-06-10

**问题描述**:
`src/app/api/flows/route.ts` 中搜索参数直接拼入 Supabase 查询的 `ilike` 表达式：
```typescript
query = query.or(`l4_process.ilike.%${search}%,process_code.ilike.%${search}%`);
```
用户输入的特殊字符（如 `%`、`_`、`'`）未经转义，可能被用于通配符注入或非预期查询。

**复现步骤**:
1. 在流程清单页搜索框输入 `%` 或 `_` 等特殊字符
2. 观察返回结果是否超出预期（如 `%` 会匹配所有记录）

**影响范围**:
- 流程清单搜索 `/functional/list`
- 修订记录搜索 `/functional/revision`

**修复方案**:
在 `src/lib/utils.ts` 中新增 `escapeLike()` 函数，对搜索关键词中的 `%`、`_`、`\` 进行转义后再传入查询。

### B007 - Next.js Turbopack 多 lockfile 警告

- **模块**: 构建配置
- **严重程度**: 低
- **状态**: 已知限制
- **发现日期**: 2026-06-10
```
⚠ Warning: Next.js inferred your workspace root, but it may not be correct.
We detected multiple lockfiles and selected the directory of /pnpm-lock.yaml as the root directory.
```

这是因为沙箱环境根目录和项目目录各有一份 `pnpm-lock.yaml`，导致 Turbopack 无法确定工作区根目录。

**建议修复**:
在 `next.config.ts` 中配置 `turbopack.root` 指定项目目录，或清理根目录多余的 lockfile。
